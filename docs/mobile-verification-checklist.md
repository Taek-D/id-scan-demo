# Mobile Verification Checklist

이 문서는 실제 Android/iPhone 기기에서 촬영 화면을 점검할 때 사용하는 체크리스트입니다.  
현재 저장소는 데모용 프로토타입이므로, 브라우저별 권한 정책과 카메라 제약은 실기기에서 반드시 다시 확인해야 합니다.

## 공통 준비

- 서버 실행: `uv run uvicorn app.main:app --reload`
- 접속 주소: 같은 네트워크에서 접근 가능한 개발용 호스트 또는 `localhost`
- 점검 대상 화면
  - Capture UI: `/`
  - Admin UI: `/admin`

## Android Chrome

- `/` 화면이 깨지지 않고 로드된다.
- `카메라 시작` 버튼 탭 시 권한 요청이 뜬다.
- 권한 허용 후 후면 카메라가 연결된다.
- 권한 거부 시 파일 업로드 fallback 메시지가 보인다.
- 신분증 종류 변경 시 프레임 라벨과 촬영 팁이 바뀐다.
- 세로/가로 전환 시 프레임 가이드와 고정 액션 버튼이 유지된다.
- 파일 업로드로도 결과 화면 진입이 가능하다.
- `홈 화면에 추가`가 가능한 브라우저 환경이면 설치 버튼이 노출된다.

## iPhone Safari

- `/` 화면이 깨지지 않고 로드된다.
- `카메라 시작` 버튼 탭 시 권한 요청이 뜬다.
- 권한 허용 후 카메라 연결 또는 파일 업로드 fallback이 동작한다.
- 권한 차단 시 안내 문구가 이해 가능하게 표시된다.
- 세로/가로 전환 후 프레임 가이드가 유지된다.
- 결과 화면에서 원본/반사광 단계/크롭/최종 보정 이미지가 보인다.
- Safari 공유 메뉴를 통한 홈 화면 추가가 가능한지 확인한다.

## 결과 기록

- 성공 여부
- 사용 브라우저와 OS 버전
- 권한 관련 이슈 여부
- 카메라 연결 실패 여부
- 파일 업로드 fallback 정상 동작 여부
- 화면 깨짐 또는 줄바꿈 이상 여부
- PWA 설치 관련 노출 여부

## 비고

- 실기기 점검은 검증용이며, 이 체크리스트 자체가 기능 구현 완료를 의미하지는 않습니다.
- 품질 판정 및 이미지 처리 결과는 조명 환경과 샘플 이미지에 따라 차이가 날 수 있습니다.
